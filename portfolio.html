Portfolio Website Case Study

Overview

The responsive portfolio website was created to showcase my work samples, skills, and design process. The website was coded in HTML, CSS, and JavaScript using design best practices. While coding the website I developed skills that will allow me to understand the technical requirements of the design and effectively collaborate with the development team.

Context and Challenge

As a UX designer, I need a way to present my skills and work samples to build a professional network. The main challenge was to build a responsive website that host my design portfolio using HTML, CSS, and JavaScript.

The project was created in five weeks. My role included designing, coding and testing the website. During this project, I learned the fundamentals of frontend development and gained coding experience.

The Process

To get a better idea of how to create an online portfolio, I looked through other professional portfolio websites. I determined the main structure of the website that consists of the home page, about page, project page, and contact section, and curated the content of each page. To customize the website, I selected the color palette and typography that represent my design style.

I translated the wireframes into HTML document and applied styling in CSS following design best practices and the mobile-first approach. Using basic JavaScript, I added interactive elements to improve the user experience of the website.

The result was a fully functioning website that was tested with five users. The issues identified in usability testing were rated by severity and the changes were suggested according to the testing results.

  • Three of five users didn't like the color palette, so I replaced it with the colors that are more appealing to users.
  • Two of five users were missing the navigation in the project page, which is why I added additional buttons for easier navigation within the description of the design process.
  • I revised the white space and reorganized the contact section based on the users' feedback.
  • The users were discontented because they need to scroll a lot to return to the top of the page, so I added â€˜back to top' button that makes this action easier.

After implementing changes, I performed cross-browser testing in all modern browsers and mobile, tablet and desktop viewports. The colors were tested for accessibility and adjusted to pass the AA contrast ratio.

The Solution

Conclusion

In the future improvements of the website, I would like to focus on adding interactivity and presenting more case studies, with continuous usability testing and iteration.

While creating a portfolio website, I gained hands-on coding skills that could help me establish effective communication with developers. Learning the basics of frontend development made me aware of the possibilities and limitations of design, which broaden my views as a designer.
